# Milestone v4.0: Workspace Records & Views

**Status:** Shipped 2026-07-15
**Phases:** 21-26
**Total Plans:** 24

## Overview

v4.0 elevated the old page-JSON Kanban placeholder into a real Records and Views engine. It added a complete generic content-block palette, a company-scoped records API, record detail pages backed by the existing page editor, a real `collection-view` block, zero-data-loss legacy Kanban migration, view controls, and six collection view types over the same `collection_records` data.

## Key Accomplishments

- Page/record bodies now have a broad generic block palette that can express structured operational notes without bespoke vertical blocks.
- Records are persisted through `data_collections`, `collection_records`, and `collection_views`, giving Vectra a real database-view foundation.
- Record detail pages reuse the existing editor rather than creating a separate record-body editor.
- `collection-view` now supports board, table, list, calendar, gallery, and timeline views without duplicating records.
- Legacy page `kanban` blocks can migrate to the new records/views model with preserved card data.

## Known Deferred Items

- Phase 25 and 26 live-browser UAT remains pending, tracked in `v4.0-MILESTONE-AUDIT.md`.
- Phase 26 has warning-level UI follow-ups around timeline geometry/copy, settings race behavior, calendar duplicate-click protection, and gallery cover URL expectations.
- Realtime record sync, CRDT co-editing, synced blocks, KPI scheduler work, program-builder gaps, OCR wiring, and event-spine retention remain out of scope for v4.0 and candidates for later milestones.
