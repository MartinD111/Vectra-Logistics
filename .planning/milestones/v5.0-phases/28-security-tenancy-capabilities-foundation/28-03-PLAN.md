<interfaces>
The public trust output should converge on a documented family such as:

```typescript
verifyTrustedPublicRequest({
  edge: 'gate-anpr' | 'gate-ocr' | 'samsara' | 'geotab',
  rawBody,
  headers,
  token,
})
```

The exact API may differ, but the important properties are:
- no unsigned tenant identity taken from the body as the trust root
- support for HMAC, shared-secret token, or signed-key models as appropriate
- reusable from multiple public-edge controllers

The demo/capability output should expose a structured policy surface, for example:

```typescript
resolveCapabilityMode(ctx, 'outlook.mail.sync')
// => { allowed: false, reason: 'unavailable', explicitFallback: null }
// or { allowed: true, explicitFallback: { kind: 'demo', label: 'Sample data' } }
```

Exact names are Claude's discretion, but production behavior must stop silently looking like live operational success when it is synthetic.
</interfaces>
