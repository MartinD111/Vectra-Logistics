<interfaces>
The dispatcher lifecycle should support a publication state machine along lines such as:

```typescript
type OutboxStatus = 'pending' | 'publishing' | 'published' | 'failed';

interface DispatchAttemptResult {
  status: 'published' | 'retry_scheduled' | 'failed';
  publishedAt?: string;
  nextAttemptAt?: string;
  lastError?: string | null;
}
```

The exact names are Claude's discretion, but the implementation must make duplicate claim/progress transitions safe even if the worker restarts or a queue delivery repeats.

The event catalog should explicitly document the first workflow-facing contract, for example:

```markdown
Event: records.collection.created
Envelope version: 1
Payload version: 1
Delivery semantics: at-least-once publication with database-enforced durable identity
Primary consumer: workflow engine (Phase 30)
Derived read model: activity_events projection for analytics/history
```

The exact doc format may differ, but those guarantees must be explicit.
</interfaces>
