<objective>
Close verification gap Truth 5 / CR-01: `main()` in `apps/api/src/scripts/install.ts` currently calls `generateSecrets()`/`upsertEnvVars()` and `createCompanyAndAdmin()` unconditionally on every invocation. `applyBaseSchema()`'s `to_regclass('public.companies')` guard only protects schema DDL, not secrets or company/admin creation. A second run silently overwrites `JWT_SECRET`/`ENCRYPTION_KEY` (permanently breaking decryption of anything encrypted under the old `ENCRYPTION_KEY`, invalidating every session) and can insert a second company+admin row. This plan adds a hard "already installed" guard (bypassable only via an explicit `--force` flag), adds the missing unit-test coverage for that guard (WR-01), and fixes the non-`Error`-safe error formatting in `main()`'s catch block (WR-02).

Purpose: Makes the installer genuinely safe to re-run per INS-01's "no manual SQL, no default credentials ... one guided flow" safety intent — closes 17-VERIFICATION.md's only failed truth and 17-REVIEW.md's CR-01/WR-01/WR-02 findings.
Output: `apps/api/src/scripts/install.ts` (new exported guard/format functions + `main()` wiring), `apps/api/src/scripts/install.test.ts` (new tests proving the guard and the error-format fix).
</objective>

1. `export async function isAlreadyInstalled(client: PoolClient): Promise<boolean>` — first runs `SELECT to_regclass('public.companies') AS exists`; if `rows[0].exists` is falsy (table does not exist — schema never applied), return `false` immediately without a second query. Comment: this is the CR-01 fix — distinguishes "schema not yet applied" from "schema applied but no company row yet" from "a real prior install exists," since only the last case must block a re-run.

2. `export function shouldBlockInstall(alreadyInstalled: boolean, force: boolean): boolean` — pure function, returns `alreadyInstalled && !force`. No I/O, trivially testable.

3. `export function formatFatalError(err: unknown): string` — returns `` `FATAL: install failed: ${err instanceof Error ? err.message : String(err)}` ``. This is the WR-02 fix — replaces the current bare `(err as Error).message` interpolation which produces `undefined` for non-`Error` throws.

Wire both into `main()`:
