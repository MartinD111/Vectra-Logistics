<objective>
Restrict CORS (Express REST API) and Socket.IO's handshake to an env-configured origin allowlist, replacing today's wide-open `cors()`/`origin: "*"` config (HRD-01).

Purpose: the production stack must reject cross-origin requests from unknown origins — this closes the CORS hole described in `docs/specs/deployment/cloud-deployment.md` §3.1.
Output: `apps/api/src/core/config/cors.ts` (new `getAllowedOrigins()` helper), `apps/api/src/server.ts` wired to it on both the Express and Socket.IO surfaces, unit + integration tests, and a new `CORS_ALLOWED_ORIGINS` var documented in `.env.example`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (any origin) → Express API | `Origin` header is untrusted, client-controlled input; determines whether CORS response headers are sent |
| Browser (any origin) → Socket.IO handshake | Independent config surface from Express `cors()`, same untrusted `Origin` input |
