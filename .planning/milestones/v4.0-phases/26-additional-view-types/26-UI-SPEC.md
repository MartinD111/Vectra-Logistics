# Phase 26 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — hand-rolled Tailwind, no `components.json` anywhere in the monorepo (confirmed by direct filesystem check this phase). Continuing the same decision Phase 25 made: the codebase has a mature hand-rolled component convention across Phases 21-25 (`BoardCard.tsx`, `BoardColumn.tsx`, `ViewSwitcher.tsx`, `ViewSettingsMenu.tsx`, `CollectionTableView.tsx`), and CLAUDE.md's "Reuse over rebuild" constraint argues against introducing shadcn mid-milestone for 4 new view renderers that are direct siblings of existing ones. Registry safety gate: not applicable. |

## Primary Visual Anchor

The records themselves remain the primary visual anchor in every new view, exactly as Phase 25 established for Board/Table — never the chrome around them:

- **Calendar**: the month grid is the anchor; the "Unscheduled" tray is secondary chrome (visually subordinate — smaller, muted background, positioned so it never outweighs the grid).
- **Gallery**: the card grid is the anchor; cover images (or their placeholders) are the dominant per-card visual element, consistent with Board's card-first hierarchy.
- **Timeline**: the bar chart area is the anchor; the month-scale header (day/week gridlines) is secondary chrome.
- **List**: the row stack is the anchor; row chrome (title + chips) carries equal weight left-to-right, no decorative dividers beyond a thin row separator.
- **ViewSwitcher dropdown**: chrome only — `primary-600` accent reserved for the selected menu row and current-view label, never used to decorate the trigger button by default (same rule as Phase 25's Color contract).

## Spacing Scale

Declared values (must be multiples of 4), matching the scale already in use across `BoardColumn.tsx`/`ViewSettingsMenu.tsx`/`CollectionTableView.tsx` (unchanged from Phase 25 — no new tokens introduced):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), chip internal padding, calendar day-cell number margin |
| sm | 8px | Compact control padding (`px-2`), List row internal padding, Gallery card internal padding, dropdown menu-item padding |
| md | 16px | Toolbar row padding, popover panel padding, Gallery grid gap (`gap-4`), Calendar grid cell padding |
| lg | 24px | Section padding between toolbar and view content (Calendar grid ↔ Unscheduled tray gap, Timeline header ↔ bars gap) |
| xl | 32px | N/A this phase (no layout-level gaps introduced beyond `lg`) |
| 2xl | 48px | N/A this phase |
| 3xl | 64px | N/A this phase |

Exceptions:
- Calendar day cells: fixed `min-h-[96px]` (not a strict 8-pt multiple of a "token" but composed of 4px multiples) to fit 2-3 stacked event chips per day without overflow — a pre-existing-style exception, same category as Phase 24's `w-56` board column width.
- Gallery card cover block: fixed `h-32` (128px) placeholder/image height, an 8-pt multiple, consistent with the scale.
- Timeline row height: `min-h-[36px]` per record lane (36px accommodates a `text-xs` label + 8px vertical padding either side; not a strict 8-pt multiple but matches the existing 32px icon-button touch-target precedent rounded up for bar legibility).
- Icon-only buttons (ViewSwitcher trigger, Calendar/Timeline prev/next nav) use the existing 32px (`min-h-[32px]`) touch target, matching `ViewSettingsMenu`'s `•••` trigger — consistent, not a new exception.

## Typography

Matches the exact type scale already declared in `25-UI-SPEC.md` and used in `BoardColumn.tsx`/`CollectionTableView.tsx` — no new sizes introduced this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 — List row title text, Gallery card-face property values, Timeline bar hover-tooltip text, Unscheduled tray record titles |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.2 — Calendar weekday headers (`uppercase tracking-wider`), Timeline bar inline label (record title truncated inside the bar), ViewSwitcher dropdown menu item labels, List row chip text |
| Heading | 12px (`text-xs`) | 600 (semibold) | 1.2 — Calendar/Timeline month label ("July 2026") in the nav header, "Unscheduled" tray section header, ViewSwitcher trigger button label |
| Display | 16px (`text-base`) | 600 (semibold) | 1.2 — Gallery card title below cover (matches existing Board/Table record-title convention, unchanged from Phase 24/25) |

Exactly 2 weights declared: 400 (regular, body text) and 600 (semibold, all label/heading/display tiers). No 700/bold weight introduced by this phase's new components — calendar day numbers use 600 semibold at 12px (Label tier), not a new bold tier.

## Color

Matches the existing Tailwind theme (`apps/workspaces/tailwind.config.ts`) — `primary` is the existing custom green scale, no new colors introduced this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `dark:bg-slate-900` | Calendar grid background, Gallery view background, Timeline chart background, List row background, ViewSwitcher dropdown panel background |
| Secondary (30%) | `gray-50` / `dark:bg-slate-800/60` (tray/card/lane backgrounds), `gray-200` / `dark:border-slate-700` (borders, grid lines) | Calendar day-cell borders + "Unscheduled" tray background, Gallery card surface + cover placeholder block, Timeline row-lane alternating background, List row separators, ViewSwitcher dropdown item hover background (`gray-100`/`dark:hover:bg-slate-800`) |
| Accent (10%) | `primary-600` (`#16a34a`) / `primary-500` dark-mode hover | Reserved for: today's date highlight on the Calendar grid (day-number background pill), the selected view row + current-view icon in the ViewSwitcher dropdown, Timeline Gantt bar fill color, focus rings (`focus:ring-primary-500`) on any new inline-editable inputs (Calendar click-to-create title field, mirrors `BoardCard`'s existing pattern) |
| Destructive | `red-600` | No new destructive actions this phase (no delete affordance introduced by any of the 4 new views — records are only created/viewed, never deleted from these surfaces). Reserved unused this phase, consistent with the "declare it even if unused" convention. |

Accent reserved for: today's-date pill (Calendar), selected-view row (ViewSwitcher dropdown), Gantt bar fill (Timeline), and focus rings on new inline inputs — never a decorative or default-state color on any of the 4 new views' chrome, continuing the exact rule from `25-UI-SPEC.md`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | ViewSwitcher trigger button: current view name + icon (e.g. "Board", "Calendar") with a `ChevronDown` caret — opens the 6-item menu ("Board", "Table", "List", "Calendar", "Gallery", "Timeline"). Calendar click-to-create: no button — clicking an empty day cell directly creates a record (inline-editable title enters edit state immediately, mirrors Board's D-07/"+ New" pattern — no separate "New event" button needed). |
| Empty state heading (Calendar, no date property chosen) | "Choose a date property to plot this calendar" — shown when `calendarDateProperty` is unset in `view.config` |
| Empty state heading (Gallery/List/Timeline, filtered to zero) | "No records match these filters" (reused verbatim from Phase 25's Table/Board empty state, for consistency across all 6 view types) |
| Empty state body (filtered to zero) | "Try removing a filter condition or adjusting a value to see more records." (reused verbatim from Phase 25) |
| Unscheduled tray heading | "Unscheduled" — section label above the tray list, Heading tier typography |
| Unscheduled tray empty state | "All records have a date" — shown when the tray list is empty (every record has the chosen date property set) |
| Timeline empty state (no start/end property chosen) | "Choose start and end date properties to plot this timeline" — shown when `timelineStartProperty` or `timelineEndProperty` is unset |
| Timeline hidden-record note | No banner/warning copy needed — per D-07 these records simply do not render on the timeline (they remain visible in other views); no "N records hidden" disclosure is required this phase (matches the "don't invent forgiving logic beyond what's asked" precedent from Phase 24's D-06) |
| Error state | "Couldn't update this view." / "Check your connection and try again." — reused verbatim from Phase 25's `useUpdateView` failure copy, applies identically to the new `calendarDateProperty`/`galleryCoverProperty`/`timelineStartProperty`/`timelineEndProperty` config writes |
| Destructive confirmation | None — no destructive actions introduced by any of the 4 new view types this phase |

## Interaction Contract (phase-specific, beyond template defaults)

Pre-populated from 26-CONTEXT.md's D-01 through D-10 so these are not re-derived during planning:

| Surface | Behavior |
|---------|----------|
| ViewSwitcher dropdown (D-10) | Replaces the existing 2-button segmented control entirely (not additive). Trigger: single button showing current view's icon + label + `ChevronDown`, same 32px touch target and hand-rolled popover shell as `ViewSettingsMenu` (`fixed inset-0 z-20` backdrop + `absolute z-30` panel, left-aligned under the trigger this time since it's the first control in the toolbar row, not the last). Menu: 6 stacked rows (icon + label), each `min-h-[32px]`, `px-2` `py-1.5`, hover `bg-gray-100 dark:hover:bg-slate-800`, selected row shows `primary-600` text/icon + a subtle `bg-primary-50 dark:bg-primary-900/20` row background. Selecting a row calls the same create-once-then-remember `switchTo(type)` logic `ViewSwitcher.tsx` already has, extended from 2 to 6 `type` values — closes the menu on select. |
| Calendar date-property picker (D-01) | New section in `ViewSettingsMenu.tsx`, shown only when `view.type === 'calendar'`: a `<select>` (same `saas-input` class as the existing aggregation-property picker) listing every `date`-type `collection.schema` property, labeled "Calendar date property" (Label tier). Persists to `view.config.calendarDateProperty` via `useUpdateView`, instant-on-change, no separate save button — same autosave convention as every other `ViewSettingsMenu` field. |
| Calendar grid (D-03) | Month grid, 7 columns (Sun–Sat weekday headers, Label tier, `uppercase tracking-wider`), `min-h-[96px]` day cells, `border` `gray-200`/`dark:border-slate-700` grid lines. Today's date cell: day-number rendered inside a `primary-600` filled circular/pill background, white text. Records plotted on their `calendarDateProperty` value render as compact chips (reuse `BoardCard`'s title + first `cardProperties[]` entry, truncated) stacked inside the day cell, click-through opens record detail (same `window.open('/collections/{id}/records/{id}', '_blank')` pattern as `BoardCard`). Prev/next month nav: two `ChevronLeft`/`ChevronRight` icon buttons flanking the "July 2026"-style month label (Heading tier), 32px touch targets. |
| Calendar click-to-create (D-03) | Clicking empty space within a day cell (not on an existing chip) calls `useCreateRecord` with `props: { [titleProp]: '', [calendarDateProperty]: <that day's ISO date> }`, then the newly created chip enters the same inline-editable title state `BoardCard` already implements (`autoFocusEdit`/`onExitEdit` props) — no navigation to record detail on creation, matches Phase 24's D-07 exactly. |
| Unscheduled tray (D-02) | Sidebar panel to the right of the calendar grid on wide viewports (`flex` row layout, tray `w-64 flex-shrink-0`), stacking below the grid on narrow viewports (`flex-col` at a Tailwind breakpoint, e.g. `lg:flex-row flex-col` — Claude's discretion per CONTEXT.md, simplest layout given the grid). Tray background `gray-50`/`dark:bg-slate-800/60`, "Unscheduled" heading (Heading tier) at top, list of record chips below (same chip rendering as calendar-plotted records — title + first card property), each click-to-open per the assumption in CONTEXT.md's discretion notes. Empty state: "All records have a date" (Body tier, `text-gray-400 italic`, matching `BoardCard`'s existing "Untitled" muted-text convention). |
| Gallery cover-property picker (D-04) | New section in `ViewSettingsMenu.tsx`, shown only when `view.type === 'gallery'`: a `<select>` listing every `files`-type `collection.schema` property, labeled "Cover image property". Persists to `view.config.galleryCoverProperty`, same autosave convention. |
| Gallery card grid (D-04, D-05) | CSS grid, responsive columns (`grid-cols-2 sm:grid-cols-3 lg:grid-cols-4`), `gap-4` (16px). Each card: `rounded-md bg-white dark:bg-slate-900 border border-gray-200 dark:border-slate-700 shadow-sm overflow-hidden` (same border/shadow/radius convention as `BoardCard`). Cover block: fixed `h-32`, first file from `galleryCoverProperty`'s value rendered as `object-cover` image when present; when absent (D-05: unset property, no property picked, or empty value) render a plain placeholder block — `bg-gray-100 dark:bg-slate-800` solid fill, no icon, no broken-image glyph. Below the cover: title (Display tier) + `cardProperties[]` (Body tier, same `formatCardPropertyValue` reuse as Board), identical to today's `BoardCard` body layout. Click-through opens record detail, same pattern as every other view. |
| Timeline start/end-property pickers (D-06) | Extends `ViewSettingsMenu.tsx` with two `<select>` fields shown only when `view.type === 'timeline'`: "Start date property" and "End date property", each listing `date`-type schema properties. Persist to `view.config.timelineStartProperty` / `view.config.timelineEndProperty` independently (two separate autosave writes, not a combined object) — matches the existing per-field `updateView.mutate` granularity already used for `columnAggregation`'s type/propId split. |
| Timeline bars (D-06, D-07, D-08) | Fixed month-wide horizontal scale: a header row of day-of-month gridlines/labels (Label tier, thin `border-l border-gray-100 dark:border-slate-800` verticals), one lane per qualifying record below (`min-h-[36px]` per lane, `space-y-1`). A record renders a bar only when both `timelineStartProperty` and `timelineEndProperty` values are set (D-07 — otherwise the record is simply absent from this view, no placeholder row, no point-marker). Bar: `bg-primary-600 rounded-md h-6` positioned/sized proportionally via inline `style={{ left, width }}` percentages computed against the visible month's day count, truncated record title inside the bar (Label tier, white text) when the bar is wide enough, else title reads to the right of the bar in default text color. Prev/next month nav: identical `ChevronLeft`/`ChevronRight` + month-label header as Calendar's (D-08 mirrors D-03's nav pattern exactly — same component/copy shape, not a new pattern). Click-through opens record detail, same pattern as every other view. |
| List rows (D-09) | Single-column stacked rows, NOT a grid — each row `flex items-center justify-between gap-3 px-2.5 py-2 border-t border-gray-100 dark:border-slate-800` (first row has no top border, container has an outer `border` matching `CollectionTableView`'s `CONTAINER_CLASS`). Left: record title, Display tier if the list is the primary content surface, else Body tier bold (`font-semibold`) — use Body tier size (14px) at 600 weight to stay within the phase's declared 2-weight typography, not the 16px Display tier (reserves Display exclusively for Gallery per the Typography table above). Right: `cardProperties[]` rendered as inline chips (`inline-flex items-center gap-1 flex-wrap justify-end`), each chip `rounded-full bg-gray-100 dark:bg-slate-800 px-2 py-0.5 text-xs text-gray-600 dark:text-gray-300`, reusing `formatCardPropertyValue`/`BoardCard`'s value-resolution logic verbatim (not Table's per-column grid layout). Click-through opens record detail, same pattern as every other view. |
| Filters/sorts (cross-cutting) | `viewFilters.ts`'s `applyFilters`/`applySorts` apply identically before records reach any of the 4 new renderers — no new toolbar UI needed beyond what Phase 25's `FilterSortToolbar` already provides; it mounts unchanged alongside the new ViewSwitcher dropdown. |
| Empty states (all 4 new views) | Centered `flex flex-col items-center justify-center py-10 text-center` block, matching `CollectionTableEmptyState`'s exact layout — heading (Body tier, `text-gray-700 dark:text-gray-300`) + body (Label-size `text-xs text-gray-400`) — reused verbatim for the "no records match filters" case; view-specific "choose a property" empty states (Calendar/Timeline when config is unset) use the same visual shell with the copy from the Copywriting Contract above. |
