# Phase 21 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled TailwindCSS components (no Radix/Base UI anywhere in `apps/workspaces`) |
| Icon library | lucide-react 0.294.0 |
| Font | Inter (`next/font/google`, `apps/workspaces/src/app/layout.tsx`) |

**Design system decision (default, not asked interactively):** No `components.json` exists anywhere in the repo, and all 20 prior phases (including the entire project-page block system this phase extends) use hand-rolled Tailwind + lucide-react with zero component-library dependency. `EditableRichText.tsx` carries an explicit code comment stating "Deliberately no editor library." Introducing shadcn now would contradict that established convention and the project's reuse-over-rebuild constraint (CLAUDE.md). Every new block in this phase MUST match the existing hand-rolled patterns below (`CalloutBlock.tsx`, `SlashMenu.tsx`, `KanbanBlock.tsx`) — no new UI dependency is introduced.

---

## Spacing Scale

Declared values (must be multiples of 4), matching existing block components exactly:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-text gaps, tight button padding |
| sm | 8px (`gap-2`, `px-2 py-1.5`) | Compact row spacing, menu item padding |
| md | 16px (`p-4`, `gap-4`) | Block container padding, popover padding |
| lg | 24px (`py-6`) | Empty-state vertical padding |
| xl | 32px (`py-8`) | Loading-state vertical padding |

**Approved Exception (2026-07-13):** `px-2.5` (10px), `py-1.5` (6px), and `gap-1.5` (6px) are kept verbatim (not rounded to the 8-point scale) for the Toggle header row, Table body cells, and Sub-page block row. Justification: visual parity with already-shipped block components (`CalloutBlock.tsx`, `SlashMenu.tsx`, `KanbanBlock.tsx`) that use these exact Tailwind classes today — rounding the new blocks to `px-2`/`py-2`/`gap-2` would create a visible, inconsistent misalignment against their existing siblings on the same page. This is a deliberate, developer-approved deviation from the 8-point scale, not an oversight, and is scoped narrowly to these three call sites.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 (Tailwind default `leading-normal`, matches `prose-sm` used in `EditableRichText.tsx`) |
| Label | 11px (`text-[11px]`) | 700 (bold/semibold) | 1.2 — used for group headers, badges, counts, table header cells, and the code-block language picker label (matches `SlashMenu.tsx`'s `GROUP_LABEL`, `KanbanBlock.tsx`'s column count) |
| Heading | 14px (`text-sm font-bold`) | 700 (bold) | 1.2 — block/section titles (matches `KanbanBlock.tsx`, `LtlMatchesBlock.tsx` title rows) |
| Code | 13px (`text-[13px]`) monospace (`font-mono`) | 400 (regular) | 1.5 — code block body text (new for CONT-04, no existing precedent; monospace is a hard requirement of a code block) |

Only 2 weights are used across this phase's new blocks: 400 (regular, body/content text) and 700 (bold, titles/labels/badges) — matches the existing codebase's exclusive use of `font-semibold`/`font-bold` for emphasis (no `font-medium` observed in block components). The label size is collapsed to a single 11px (`text-[11px]`) across every use case (group headers, badges, counts, table headers, code-block language picker) — a separate 10px size is not needed anywhere in this phase's block set.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-white` / `dark:bg-slate-900` | Block container backgrounds, popover backgrounds |
| Secondary (30%) | `bg-gray-50` / `dark:bg-slate-800` | Nested/inset surfaces: table header row, column lanes, icon swatch backgrounds, code block background |
| Accent (10%) | `bg-primary-600` (`#16a34a`) / `text-primary-600` | New-block insert buttons, active/selected slash-menu row (`bg-primary-50`/`dark:bg-primary-900/20`), toggle expand chevron on hover, mention span text color, "Add sub-page" CTA |
| Destructive | `text-red-500` / `bg-red-100` `text-red-700` (dark: `bg-red-900/30` `text-red-400`) | Delete-item icon on checklist rows and table row/column delete controls only |

Accent reserved for: primary CTA buttons ("Add column", "Add sub-page", language-picker active state), the active row highlight in the `@mention` popover (mirroring `SlashMenuPanel`'s `bg-primary-50` active state), toggle chevron hover state, and the rendered `@mention` span's text/underline color. Never apply accent to static content text, borders on non-interactive containers, or disabled elements.

**Visual hierarchy on a mixed-block page:** across any page containing a mix of this phase's new blocks, the accent-colored elements (active slash-menu row, toggle chevron hover, mention span text, primary CTAs) are the ONLY intentional visual anchor — every other surface, border, and text color stays neutral gray/slate so the accent reads as a clear, singular focal point rather than competing with it.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add sub-page" (sub-page block creation button); "Add column" (table); slash-menu item titles: "To-do list", "Toggle list", "Quote", "Code", "Image", "File", "Video", "Bookmark", "Embed", "Table", "Columns", "Sub-page" |
| Empty state heading | Table: "No rows yet" · Columns: "+ Add block" per lane · Toggle: collapsed-with-no-children shows no special copy (chevron simply has nothing to expand) |
| Empty state body | Media block unconfigured: "Paste a link or drop a file" · `@mention` popover empty query/no match: "No matching people, pages, or dates." |
| Error state | Media URL validation failure: "That doesn't look like a valid link — check the URL and try again." (inline, below the URL input, `text-xs text-red-500`) |
| Destructive confirmation | Deleting a checklist item / table row / table column: no confirmation dialog (matches existing low-stakes inline-delete pattern used by `KanbanBlock.tsx`'s card delete — instant, undo via normal page-edit history). Deleting an entire block (any kind, via the existing block toolbar) reuses `LivePageCanvas.tsx`'s existing delete-block affordance and its existing confirmation behavior — unchanged by this phase. |

## Nesting & Interaction Contract (new ground this phase establishes)

| Concern | Contract |
|---------|----------|
| Toggle collapse/expand | Click anywhere on the header row (icon + title) toggles `collapsed`. Chevron rotates 90° via `transition-transform` (`ChevronRight` → rotated when expanded, no icon swap, to avoid layout jump). Collapsed state persists in `PageConfig` JSON (`collapsed: boolean` field), not just client UI state. |
| Nested child indentation | `pl-5` (20px) indent + `border-l border-gray-100 dark:border-slate-800` rail, applied uniformly to both toggle children and column-lane children — one shared visual treatment per D-01's "one mechanism" discretion choice. |
| Nested child insertion | "+ Add block" ghost row at the bottom of the nested list (style matches `KanbanBlock.tsx`'s "+ Add card" button: `text-xs text-gray-400 hover:text-gray-600`), opens a slash-style menu filtered to `nestable: true` items only (new flag on `PageBlockDef`, per Pitfall 5) — never the full palette. |
| Nested child reordering | No drag-and-drop inside nested children this phase (per D-01/Pitfall 3) — ship with no reorder control, or optional up/down icon-buttons (`ChevronUp`/`ChevronDown`, 12px, `text-gray-300 hover:text-gray-500`) if trivial to add. Do not attempt a second `@dnd-kit` `SortableContext` inside nested lists. |
| Nesting depth enforcement | UI-level: the nested "+ Add block" menu never lists `toggle` or `columns` as options (enforces D-02 visually, not just at the data layer). |
| Column count | Fixed 2 columns (`grid grid-cols-2 gap-4`), no add/remove-lane UI this phase (D-04 cheapest option). |
| `@` mention popover positioning/behavior | Identical mechanics to `/` slash menu: two-phase trigger detection, live-filtered list, `Enter`/click to commit, `Escape` to dismiss — reuse `SlashMenuPanel`'s exact container styling (see table above), only the data source and grouping differ. |
| Mention click behavior | `@person` → navigate to person profile/detail (or hover card) · `@page` → open page in new tab via existing `crossAppUrl` convention · `@date` → no-op, styled text only (all per locked decision D-10). |
| Sub-page block click behavior | Click anywhere on the row navigates to the created sub-page (new tab, consistent with D-13's "static title + icon row, click navigates"). |
