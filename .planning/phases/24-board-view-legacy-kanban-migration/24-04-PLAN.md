<objective>
Fix two confirmed wiring bugs from 24-VERIFICATION.md that block BOARD-01 and BOARD-03:

1. `BoardBlock.tsx`'s D-04 zero-picker board provisioning is permanently stuck on "Creating board..." because `useCreateView(block.collectionId ?? '')` binds its mutation's target collection id at hook-construction time (when `block.collectionId` is still `null`), not at call time — so `createView.mutateAsync(...)` always POSTs to `/collections//views` (404, silently swallowed by `.catch(console.error)`).
2. `useCreateRecord` in `useRecords.ts` has no `onSuccess` cache write, so a card created via BoardColumn's "+ New" button is persisted server-side but invisible on the board for up to 5 minutes (query `staleTime: 5min`, `refetchOnWindowFocus: false`), breaking BOARD-03's "immediately shows an autofocused inline-editable title" acceptance criterion.

Purpose: Unblock the two primary user-facing board flows (creating a new board from the slash menu, creating a new card inline) that are the actual point of Phase 24 — everything else in the phase (drag-and-drop, legacy kanban migration) already works.
Output: Both flows functional and directly observable without waiting on cache expiry or hitting a 404.
</objective>
