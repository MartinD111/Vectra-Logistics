## Spacing Scale

Declared values (must be multiples of 4), matching `KanbanBlock.tsx` and Phase 23's property panel exactly:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, card internal gaps (`gap-1`, `gap-1.5`) |
| sm | 8px | Column header padding, card stack spacing (`space-y-1.5` → rounded to `sm`/`md` mix), `gap-2` |
| md | 16px | Board block outer padding (`saas-card !p-4`), inter-column gap (`gap-3` ≈ rounded to this band), section spacing |
| lg | 24px | Reserved for board-block-to-page-content spacing (matches other blocks' vertical rhythm) |
| xl | 32px | Not used at board-block scale this phase |
| 2xl | 48px | Not used at board-block scale this phase |
| 3xl | 64px | Not used |

Exceptions:
- Column width: `w-56` (224px) — not on the 8-point scale, but this is the existing `KanbanBlock.tsx` convention; carry forward unchanged for the new `BoardColumn`, do not invent a new width.
- Card drag touch target: minimum **44px** effective height on the card's draggable surface (via `useSortable`'s `listeners` spanning the full card, not just a grip icon) — required so `PointerSensor`'s `activationConstraint: { distance: 8 }` has a comfortably large hit area on touch devices, consistent with WCAG minimum even though desktop-first.
- Inline "+ Add column" control and "+ New" card control: `min-h-[32px]` (not 48px) — these are secondary/low-frequency actions, not primary data-entry rows like Phase 23's property panel; keep them visually lightweight to match `AddCardInput`'s existing collapsed-button size (`px-1.5 py-1.5`).

## Typography

Inherits Phase 23's two-weight system exactly — no new sizes or weights introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (card title text) | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label (column header, "N cards" count) | 10–12px (`text-[10px]`/`text-xs`, uppercase, `tracking-wider` for column title) | 700 (bold) | 1.5 |
| Heading (board block title, if set) | 14px (`text-sm font-bold`, matches existing `KanbanBlock.tsx` block-title convention — this is a block-level label, not a page heading) | 700 (bold) | 1.2 |
| Display | not used this phase | — | — |

**Weight consolidation:** Exactly 2 weights used: **400 (regular — card body text, input text)** and **700 (bold — column headers, card count badges, block title, "+ Add column"/"+ New" action labels)**. This matches Phase 23's precedent exactly; do not introduce 500/600 anywhere in board UI.

## Color

Inherits the existing `primary` (green) scale and `dark:` conventions from `tailwind.config.ts` — no new colors introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `dark: #0f172a` (`bg-white` / `dark:bg-slate-900`) | Card surface background |
| Secondary (30%) | `#f9fafb` / `dark: #1e293b`-ish (`bg-gray-50` / `dark:bg-slate-800/60`) | Column background (existing `KanbanBlock.tsx` convention), board block wrapper (`.saas-card`) |
| Accent (10%) | `#16a34a` (`primary-600`) / focus ring `primary-500` | "+ Add column" action, "+ New" card action, drag-active card border/ring, focus ring on inline-editable inputs (column title rename, new-card title) |
| Destructive | `#dc2626` (`red-600`) | Blocked-delete-column warning text/icon (D-06: disabled trash icon or inline warning, e.g. "Move cards out first"), remove-card `X` hover state (existing `hover:text-red-500` pattern carried from `KanbanBlock.tsx`) |

Accent reserved for: the "+ Add column" control, the "+ New" inline card-creation control, the focus-visible ring on column-title-rename and new-card-title inputs, and the drag-overlay/active-card outline while dragging (`ring-2 ring-primary-500`). Accent is NOT used for column headers, card backgrounds, or passive text — mirrors Phase 23's strict reservation list.

**Migration toast (D-02):** Uses the existing success-banner color pair (`bg-green-50 dark:bg-green-900/20 border-green-200 dark:border-green-800 text-green-800 dark:text-green-300`, from `profile/page.tsx`'s toast pattern) — NOT `primary-600` accent green, to keep the semantic distinction between "brand accent action" and "success/status notification" consistent with existing usage.

## Layout & Interaction Contract

(Supplemental section — captures composition decisions RESEARCH.md flagged as needing to be locked before planning.)

**Primary visual anchor:** The card being dragged (drag overlay / active card) is the sole element with an accent-colored ring during any interaction — everything else on the board is neutral gray/white until touched.

| Element | Contract |
|---------|----------|
| Board block wrapper | `saas-card !p-4` — exact same wrapper `KanbanBlock.tsx` already uses; do not introduce a new block chrome |
| Column | `w-56 flex-shrink-0 rounded-lg bg-gray-50 dark:bg-slate-800/60 p-2` — unchanged from `KanbanBlock.tsx`; column header row: title (label style, inline-editable on click per D-05) + card count badge (`text-[10px] text-gray-400`) + optional "+ Add column" trigger only on the trailing/last column slot |
| Column header rename (D-05) | Click column title → becomes an inline `<input>` (same visual weight as the static label, `text-xs font-semibold uppercase tracking-wider`), blur or Enter commits via `PATCH /collections/:id` (schema PATCH, updates the option's `label` only — never its `id`, per RESEARCH.md Pitfall 4), Escape cancels |
| Add column (D-05) | Rendered as a narrow `w-56 flex-shrink-0` ghost column at the end of the row (dashed border, `border-2 border-dashed border-gray-200 dark:border-slate-700`, centered "+ Add column" text) that becomes an inline text input for the new column's label on click; commits by appending a new `{id: uid(), label}` option via schema PATCH |
| Card | `rounded-md bg-white dark:bg-slate-900 border border-gray-200 dark:border-slate-700 px-2.5 py-2 text-sm shadow-sm` — unchanged visual shell from `KanbanBlock.tsx`; entire card surface is the `useSortable` drag handle (no separate grip icon needed at 224px column width — matches existing whole-card interaction model, not a new pattern) |
| Card while dragging (drag overlay) | Same shell + `ring-2 ring-primary-500 shadow-lg opacity-90` — the one accent-colored state on the whole board |
| Card click-to-open | Opens `/collections/[collectionId]/records/[recordId]` via `window.open(url, '_blank', 'noopener,noreferrer')` — same-app new tab, NOT `crossAppUrl` (per RESEARCH.md, this route lives inside `workspaces` itself). Click-to-open must be disambiguated from drag-start (only fire navigation if pointer didn't move past the `PointerSensor`'s `activationConstraint: { distance: 8 }` threshold — `@dnd-kit`'s own click-vs-drag disambiguation handles this natively via `listeners`, no extra logic needed) |
| New card inline creation (D-07, BOARD-03) | Click "+ New" in a column → `createRecord` fires immediately (empty title, groupBy pre-set) → new card renders instantly with its title in inline-editable state (same visual affordance as the column-rename input: full-width text `<input>` inside the card shell, autofocus, no border-radius change). Blur or Enter commits the title via `PATCH /records/:id` (800ms debounce while typing, same as Phase 23's `InlineTextField`). Escape does NOT delete the just-created record (record already persisted server-side) — it just blurs/exits edit mode, leaving the record titled "Untitled" until edited again. Stays on the board — never auto-navigates to the record detail page |
| Blocked column delete (D-06) | Delete affordance (trash icon, appears on column-header hover) is disabled (`opacity-40 cursor-not-allowed`, no hover color change) whenever `column.cards.length > 0`; hovering/clicking a disabled delete shows the warning copy above as a small tooltip-style inline text, not a modal |
| Drag-and-drop mechanics | `@dnd-kit/sortable` multi-container pattern per RESEARCH.md Pattern 1 — one `DndContext` at board root, one `SortableContext` per column (`verticalListSortingStrategy`), `PointerSensor` with `activationConstraint: { distance: 8 }` (matches `dispatch/page.tsx`'s existing sensor config). `onDragEnd` fires a single `PATCH /records/:id` with both `props` (new groupBy value) and `sort_order` |
| Migration toast (D-02) | Fixed-position, bottom-right (`fixed bottom-6 right-6 z-50`), auto-dismiss after 4000ms, success-green styling per Color section above, no manual dismiss button needed (ephemeral, fire-and-forget per D-02) — slides/fades in, no persistent state tracked |
| Legacy kanban rendering (pre-migration) | Unchanged — `KanbanBlock.tsx`/`KanbanBoardView` continues to render exactly as today until the first edit triggers migration; no visual changes to the legacy renderer itself |
| Touch targets | Card drag surface: effectively the full card (≥44px height in practice given `py-2` + text line height); "+ Add column"/"+ New" controls: `min-h-[32px]` per Spacing Scale exception above |
