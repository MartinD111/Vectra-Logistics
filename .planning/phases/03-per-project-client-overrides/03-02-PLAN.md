- **Critical correctness rule (RESEARCH.md Pitfall 2):** every save (both "commit an override edit" and "reset to default") must send ALL THREE override fields in the `LinkProjectInput` body, not just the one being changed — read the current override state for the other two fields before constructing the mutation payload, or the `ON CONFLICT DO UPDATE` upsert will silently null out unrelated fields. Concretely: maintain the "current override intent" for a link as three independent values (`override_rate_eur`, `override_responsible_employee_id`, `override_notes`, each `number|string|null | undefined` for "not overridden") derived from `is_overridden` + the merged value, and always submit the full triple via `useUpsertClientProjectLink(clientId).mutate({ project_id, override_rate_eur, override_responsible_employee_id, override_notes })` regardless of which single field the user just touched.
