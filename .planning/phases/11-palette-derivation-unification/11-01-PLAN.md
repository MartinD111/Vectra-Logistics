<objective>
Extract one shared, registry-driven `buildPaletteItems(registry)` helper in `apps/workspaces/src/lib/workspaceEngine/palette.ts`, and make both the project-page slash/insert menu (`lib/projectPage/slashMenu.ts`) and the mini-program builder's "Add block" palette (`components/miniProgram/MiniProgramBuilder.tsx`) derive their base item lists from it — replacing the mini-program add-menu's direct iteration over the raw `BLOCK_REGISTRY` array and the page slash menu's ad-hoc `pageBlockRegistry.list()` + manual `available` filter.

Purpose: Satisfies PAL-02 — a new `WorkspaceBlockPlugin` registry entry (native, in either `pageBlockRegistry` or `miniProgramBlockRegistry`) must appear in its palette with zero edits to menu components, closing the last gap in the v2.0 engine-unification work (Phases 7-10 unified rendering/editing; this phase unifies palette derivation).
Output: `lib/workspaceEngine/palette.ts` (new), updated `lib/workspaceEngine/index.ts`, `lib/projectPage/slashMenu.ts`, `components/miniProgram/MiniProgramBuilder.tsx`.
</objective>

<!-- From apps/workspaces/src/lib/workspaceEngine/types.ts -->
```typescript
export interface WorkspaceBlockPlugin<B, Ctx> {
  key: string;
  source: PluginSource;       // 'native' | 'manifest'
  group: string;
  title: string;
  description: string;
  icon: string;                // lucide icon name

  create: () => B;
  renderer?: ComponentType<BlockRenderProps<B, Ctx>>;
  editor?: ComponentType<BlockEditProps<B, Ctx>>;
  settings?: ComponentType<BlockSettingsProps<B, Ctx>>;
  manifest?: PluginBlockManifest;
}
```

```typescript
export class WorkspaceBlockRegistry<B, Ctx> {
  get(block: B): WorkspaceBlockPlugin<B, Ctx> | undefined;
  list(): WorkspaceBlockPlugin<B, Ctx>[];
  keys(): string[];
  render(block: B, ctx: Ctx): ReactNode;
  renderEditor(block: B, ctx: Ctx, onUpdate: (block: B) => void): ReactNode;
}
```

```typescript
export const pageBlockRegistry = new WorkspaceBlockRegistry<PageBlock, PageCtx>(entries, (block) => block.kind);
```

<!-- miniProgramBlockRegistry — apps/workspaces/src/lib/miniProgram/registry.tsx -->
```typescript
export const miniProgramBlockRegistry = new WorkspaceBlockRegistry<Block, MiniProgramCtx>(
  entries,
  (block) => (block.kind === 'plugin' ? (block as PluginBlockInstance).pluginId : block.kind),
  (block) => <DynamicBlockView block={block as PluginBlockInstance} />,
);
```
Note: `miniProgramBlockRegistry`'s static `entries` cover every `BlockKind` except `'plugin'` — `.list()` returns exactly the native (non-plugin) blocks, i.e. the same set currently sourced from `BLOCK_REGISTRY` in `lib/miniProgram/blocks.ts`. Runtime-installed manifest plugins (`usePlugins()` from `lib/miniProgram/plugins/registry.ts`) are NOT static registry entries and are out of scope for this helper — `MiniProgramBuilder.tsx` keeps rendering its separate "Plugins" section unchanged.
