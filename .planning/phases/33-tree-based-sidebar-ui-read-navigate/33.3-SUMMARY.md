# Phase 33 Plan 3: Breadcrumbs Ancestor-Path Component Summary

**Pure DFS `findPath()` over the cached full tree plus a reusable `Breadcrumbs` component rendering a ChevronRight-separated, ID-keyed-link ancestor chain ending in a plain current-segment label.**

## Accomplishments
- `findPath(nodes, targetId, trail?)` walks `TreeNode[]` depth-first, returning the root-to-target ancestor chain or `null` when not found or given an empty tree — verified via 4 assert-based smoke tests run through `npx tsx`
- `Breadcrumbs` component consumes the shared `useFullTree()` cache (no new network request) and `treeNodeUrl()` (no new route scheme), rendering ancestors as clickable links and the current node as plain semibold text
- Both loading and not-found states render `null` — no skeleton, no broken-breadcrumb flash — matching UI-SPEC's interaction contract and the T-33-06 threat disposition (avoid leaking node existence via distinguishable error states)
