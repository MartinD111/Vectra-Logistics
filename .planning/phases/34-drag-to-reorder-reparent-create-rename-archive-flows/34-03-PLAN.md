```
function ArchiveConfirmDialog(props: {
  node: TreeNode;                 // must be node_type === 'folder' | 'project'
  onClose: () => void;
  onArchived: (node: TreeNode, totalDescendants: number) => void;
}): JSX.Element

function TreeUndoToast(props: {
  nodeName: string;
  onUndo: () => void;
  onDismiss: () => void;
}): JSX.Element
```

`ArchiveConfirmDialog` performs the archive mutation itself on confirm (mirrors `FolderModal.tsx`'s submit-mutates-directly pattern) — it does NOT ask the caller to pass a mutation function in. It picks `useArchiveFolder()` or `useArchiveProject()` internally based on `node.node_type`. After a successful mutation it calls `onClose()` then `onArchived(node, totalDescendants)`, where `totalDescendants` is the sum of all `countDescendants(node)` values — the caller (Wave 3's `TreeSection.tsx`) uses this to decide whether to show `TreeUndoToast` (only when `totalDescendants === 0`, per D-07).

`TreeUndoToast` renders unconditionally when mounted — the caller is responsible for the zero-descendant gating and for unmounting it after `onDismiss`/`onUndo` fires or after its own internal timer elapses.
