<!-- Component contract downstream Wave 2 plans (34-04) will consume verbatim. -->

```
type TreeContextMenuAnchor =
  | { type: 'button' }
  | { type: 'point'; x: number; y: number };

interface TreeContextMenuAction {
  id: string;
  label: string;
  icon: LucideIcon;       // from 'lucide-react'
  destructive?: boolean;  // true only for "Archive" (D-09)
  onSelect: () => void;
}

function TreeContextMenu(props: {
  anchor: TreeContextMenuAnchor;
  actions: TreeContextMenuAction[];
  onClose: () => void;
}): JSX.Element
```

The component renders ONLY the backdrop + panel (not a trigger button) — the caller (`TreeNodeRow.tsx`/`TreeSection.tsx` in Wave 2) owns the trigger element (kebab button or right-click handler) and the open/close boolean state, and conditionally mounts `<TreeContextMenu>` when open. For `anchor.type === 'button'`, the caller MUST wrap both its trigger button and `<TreeContextMenu>` in a `<div className="relative">` (matching `ViewSettingsMenu.tsx`'s shell) so the panel's `absolute right-0 top-full mt-1` positioning resolves correctly. For `anchor.type === 'point'`, no wrapping `relative` div is needed — the panel positions itself via `fixed` + inline `left`/`top` styles.
