## Spacing Scale

Declared values (must be multiples of 4), matching existing sidebar/tree/popover code:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps inside menu items, chevron rotation offsets |
| sm | 8px | Row internal gaps (`gap-3` ≈ 12px is the existing tree-row convention — see exception below) |
| md | 16px | Per-depth tree indent step (`depth * 16`, matches `TreeNodeRow.tsx` line 46 exactly — do not change this value) |
| lg | 24px | Context menu / dialog internal padding |
| xl | 32px | Modal outer padding on small viewports |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Existing tree rows use `gap-3` (12px) and `px-3 py-2` (12px/8px) — this is the established `TreeNodeRow.tsx` convention (source of truth, `apps/workspaces/src/components/tree/TreeNodeRow.tsx:31`). New drag/kebab/menu affordances added to the row MUST reuse these exact values, not the abstract 8px scale, to avoid visual drift within the same row.
- Kebab button touch target: `min-h-[32px] min-w-[32px]` (existing `ViewSettingsMenu.tsx` convention) — below the ideal 44px touch target but consistent with the rest of this desktop-oriented sidebar; do not introduce a larger target that breaks row alignment.
- Drag-reorder line indicator: 2px height, full row width — a deliberate sub-4px exception for a hairline visual affordance (not a spacing token, a border/divider treatment).

## Typography

Pre-populated from existing codebase conventions (`TreeNodeRow.tsx`, `ViewSettingsMenu.tsx`, `FolderModal.tsx`) — this phase does not introduce new type scale, it extends the existing tree row with new interactive elements that must match.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (tree row label, menu item, dialog body text) | 14px (`text-sm`) | 500 (`font-medium`) for tree rows / 400 (`font-normal`) for menu items and dialog body | 1.5 |
| Label (section headers, e.g. "Card properties" style micro-labels, uppercase group labels) | 10px (`text-[10px]`) | 700 (`font-bold`, `.label-xs` utility) | 1.4 |
| Heading (dialog/modal titles, e.g. "Archive folder?") | 16px (`text-base`, matches `FolderModal.tsx`'s `<h2>`) | 700 (`font-bold`) | 1.2 |
| Display | not used this phase (no page-level heading introduced) | — | — |

Declared exactly 3 sizes in active use (14px, 10px, 16px) and exactly 2 weights in active use (400/500 regular-ish body, 700 bold) — matches existing codebase discipline; the 500-weight tree-row label is treated as the body role's default per existing `TreeNodeRow.tsx` (`font-medium` on the row wrapper), not a third weight, since menu/dialog net-new text uses 400.

## Color

Pre-populated from `tailwind.config.ts` (`primary` = logistics green `#22c55e`/`#16a34a`, `dark.bg`/`dark.card`/`dark.border`) and existing component usage (`ViewSettingsMenu.tsx`, `KanbanMigrationGate.tsx`, `FolderModal.tsx`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `dark:bg-slate-900` (`bg-white dark:bg-slate-900`) | Sidebar background, context menu panel background, dialog background |
| Secondary (30%) | `gray-50`/`gray-100` / `dark:bg-slate-800` (hover row background, menu item hover) | Row hover state, kebab button hover, menu item hover, `dark.card` (`#1e293b`) for modal cards |
| Accent (10%) | `primary-600` `#16a34a` (light) / `primary-400` `#4ade80` (dark) | RESERVED FOR: active tree-row highlight (already `bg-primary-50 dark:bg-primary-900/20 text-primary-700 dark:text-primary-400`), drag-drop line indicator color, "into folder" reparent highlight ring, primary confirm button in dialogs (e.g. "Create"), Undo-toast action link text |
| Destructive | `red-600` (light) / `red-400` (dark) — no existing red token in `tailwind.config.ts`, use Tailwind's built-in `red-*` scale as the codebase does elsewhere for error states | RESERVED FOR: Archive confirmation dialog's confirm button ("Archive"), illegal-drop error toast/message, delete-adjacent destructive menu item text (the "Archive" menu item label itself, not other menu items) |

Accent reserved for: active/selected tree row background+text, drag reorder-line + reparent-highlight indicators, primary "Create"/"Confirm" buttons in the new dialogs, Undo-toast link. Never applied to default/inactive rows, non-destructive menu items, or body text.

## Copywriting Contract

Pre-populated from `34-CONTEXT.md` (D-05, D-09, D-10) and `34-RESEARCH.md`'s exact server error strings — TREEOPS-04 explicitly requires verbatim server error text, not paraphrase.

| Element | Copy |
|---------|------|
| Primary CTA (context menu items) | "New Folder" / "New Project" / "Rename" / "Archive" — exactly these 4 labels per D-09, no additional actions |
| Primary CTA (Archive dialog confirm button) | "Archive" (folder) / "Archive" (project) — action-specific, not a generic "Confirm" |
| Empty state heading | Not applicable this phase — tree emptiness/loading states were already specified in Phase 33's UI-SPEC; this phase adds no new empty-list state, only an always-visible "+" root-create affordance next to the "Workspace" section header (per Research Open Question 2 recommendation) |
| Empty state body | N/A (see above) |
| Error state — illegal drop (cycle) | Surface verbatim: **"Cannot move a folder into its own descendant"** (exact server string from `folders.service.ts` `moveFolder`, per D-04/TREEOPS-04 discretion note — do not paraphrase) |
| Error state — illegal drop (depth limit) | Surface verbatim: **"Folder nesting cannot exceed depth 3"** (exact server string) |
| Error state — permission (403, non-admin) | Fallback for any `ApiError` without a specific 400 body: **"You don't have permission to do this."** — per Pitfall 5, must not be a raw JSON dump or unhandled rejection |
| Error state — generic/network failure | Fallback: **"Something went wrong moving this item."** — only for non-`ApiError` failures (network errors), per `34-RESEARCH.md`'s code example |
| Archive confirmation dialog body (with descendants) | **"This will also archive {N} folders, {N} projects, {N} programs, {N} collections, {N} pages."** — per-type breakdown (D-05), omit any type with a zero count rather than showing "0 folders" |
| Archive confirmation dialog body (leaf, zero descendants) | **"This folder/project has no contents and will be archived."** (or equivalent single-line reassurance — no descendant list needed for zero-count case) |
| Destructive confirmation — Archive | Archive: dialog title **"Archive {node.name}?"**, confirm button **"Archive"**, cancel button **"Cancel"** — same `ArchiveConfirmDialog` component for both folders and projects (D-06) |
| Undo toast (zero-descendant archive only) | **"{node.name}" archived.** with an inline **"Undo"** link-button (per D-07; toast/Undo must NOT appear when the archived node has descendants) |
| Inline rename — empty/unchanged input | No error copy shown — silently revert to the prior name on blur/Escape if the trimmed value is empty or unchanged (matches `BoardColumn.tsx`'s `commitRename` behavior, no toast/validation message needed for this case) |

Destructive actions in this phase: **Archive** (folder or project) is the only destructive action. Confirmation approach: modal dialog (`ArchiveConfirmDialog`, reusing `FolderModal.tsx`'s `fixed inset-0 z-50 flex items-center justify-center bg-black/30 backdrop-blur-sm` shell) showing the descendant-count breakdown before the mutation fires — never a bare browser `confirm()`.
