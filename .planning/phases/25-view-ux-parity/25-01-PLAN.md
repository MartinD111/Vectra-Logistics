    `export const FILTER_OPERATORS: Record<CollectionPropertyDef['type'], { value: string; label: string }[]>` mapping each of the 12 property types to its valid operators:
    - `text`, `url`, `email`, `phone`: `equals` ("Is"), `contains` ("Contains")
    - `number`: `equals` ("="), `gt` (">"), `lt` ("<"), `between` ("Between")
    - `date`: `equals` ("On"), `before` ("Before"), `after` ("After"), `between` ("Between")
    - `select`, `person`: `is` ("Is"), `is_not` ("Is not")
    - `multi-select`, `files`, `relation`: `contains` ("Contains"), `not_contains` ("Does not contain")
    - `checkbox`: `is` ("Is")

    `export function evaluateCondition(record: CollectionRecord, condition: FilterCondition, schema: CollectionPropertyDef[]): boolean` — looks up the property by `condition.propId` in `schema`, reads `record.props[condition.propId]`, and evaluates per `condition.operator`:
    - `equals`: `String(recordValue ?? '') === String(condition.value ?? '')` (numbers compared with `Number(recordValue) === Number(condition.value)` when `property.type === 'number'`)
    - `contains`/`not_contains`: for array-valued props (`multi-select`/`files`/`relation`), `Array.isArray(recordValue) && recordValue.includes(condition.value)`; for string props, case-insensitive substring match
    - `gt`/`lt`: `Number(recordValue) > Number(condition.value)` / `<`
    - `between`: `condition.value` is `[min, max]`, checks `recordValue >= min && recordValue <= max` (numeric or date-string comparison via `new Date(...)` for `date` type)
    - `before`/`after`: date-string comparison via `new Date(recordValue) < new Date(condition.value)` / `>`
    - `is`/`is_not`: strict equality (`recordValue === condition.value`) / negation — for `checkbox`, `condition.value` is boolean
    - Unknown operator or missing property: return `true` (fail-open, never hide a record due to a malformed condition)

    `export function applyFilters(records: CollectionRecord[], filters: FilterCondition[], schema: CollectionPropertyDef[]): CollectionRecord[]` — per D-02, AND-only: `if (!filters.length) return records; return records.filter((r) => filters.every((f) => evaluateCondition(r, f, schema)));`

    `export function applySorts(records: CollectionRecord[], sorts: SortCondition[], schema: CollectionPropertyDef[]): CollectionRecord[]` — if `!sorts.length` return `records` unchanged (preserve existing `sort_order`); otherwise return a new sorted array (`[...records].sort(...)`) applying sort conditions in array order (first condition primary, subsequent as tiebreakers), reading `a.props[propId]`/`b.props[propId]`, comparing numerically for `number` properties, via `Date` for `date` properties, and via `String(...).localeCompare(...)` otherwise; multiply comparison by `-1` when `direction === 'desc'`.

''])`.filter to `typeof v === 'number'`, then `sum` = `.reduce((a,b) => a+b, 0)`, `avg` = `sum / values.length` (return `0` if `values.length === 0`, never `NaN`).
