<interfaces>
By the end of this plan, pilot controllers should be able to use shared helpers along the lines of:

```typescript
const ctx = requireRequestContext(req);
const companyId = requireCompanyId(ctx);
assertCapability(ctx, 'record.write', { collectionId });
```

The exact helper names are Claude's discretion, but the rollout must eliminate controller-local request/company helper duplication across the pilot set and make capability checks reusable from those domains.

Repository/service contracts should remain company-scoped. Where repository helpers are currently unsafe by shape (for example, raw `findById` methods without `company_id`), this plan may either:
- tighten the helper signature directly, or
- keep the helper but pin it behind stronger service/controller contracts and tests

The result must be testable for cross-tenant negative behavior in Plan 28-04.
</interfaces>
