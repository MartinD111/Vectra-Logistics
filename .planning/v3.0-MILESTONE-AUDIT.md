# v3.0 On-Premise GA — Milestone Audit

**Audited:** 2026-07-13
**Status:** passed (2 blockers found during cross-phase integration check, fixed inline before this report was finalized)

## Summary

All 7 phases (14-20) individually verified — every phase has a VERIFICATION.md
with status `passed`. Phase 18 was discovered mid-audit to be missing its
VERIFICATION.md entirely (executed and marked complete, but never
retroactively checked); it was verified now (4/4 must-haves, no gaps) and
committed as part of this audit.

All 17 v1 requirements are satisfied. INS-01/INS-02 were flagged `[ ]`
unchecked / "Pending" in REQUIREMENTS.md despite Phase 17's own VERIFICATION.md
marking them `✓ SATISFIED` — a documentation-sync gap the phase 17 verifier
explicitly called out as "non-blocking, typically updated at milestone close."
Synced now.

The cross-phase integration checker found **2 genuine blockers** that no
single phase's own verification could have caught, since each phase only
tests its own module in isolation:

1. `docker-compose.prod.yml`'s `api` service never forwarded
   `CORS_ALLOWED_ORIGINS`/`NEXT_PUBLIC_*_URL` (HRD-01) or `TRUST_PROXY_HOPS`
   (phase 20's post-review CR-01 fix) into the container. In the actual
   shipped compose file, the CORS allowlist always resolved empty (blocking
   all three frontends in production) and the CR-01 fix's env var never
   reached the container — reproducing the exact rate-limiter 500 bug the
   fix commit was written to close.
2. No shipped documentation (`docs/DEPLOYMENT.md`, `README.md`) told an
   operator that the one-shot installer (`apps/api/src/scripts/install.ts`,
   Phase 17) exists or how to run it before starting the stack — despite
   this being the exact mechanism the milestone's core value ("install...
   without Vectra staff present") depends on.

Both were fixed inline during this audit (commit `981b53d`): the compose
file now forwards all four env vars (defaulting the CORS origins to the
same `NEXT_PUBLIC_*_URL` vars already required by the frontend services, and
`TRUST_PROXY_HOPS` to `0`, fail-closed), and `docs/DEPLOYMENT.md` gained a
"Fresh install" section walking through installer invocation end-to-end.
Full `apps/api` build + test suite (90/90) re-run clean after the fix.
